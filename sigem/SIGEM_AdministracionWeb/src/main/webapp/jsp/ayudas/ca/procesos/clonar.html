Pasos a seguir para realizar una clonación completa:

1) Para clonar una entidad pulse sobre el botón Clonar de la entidad correspondiente del listado de entidades.
2) Introducir los datos de la nueva entidad:



  • Nombre Corto: Descripción corta de la entidad (obligatorio)
  • Nombre Largo: Descripción larga de la entidad (obligatorio)
  • Callejero: Asociar la entidad al Sistema de Información Territorial


    • 3) Introducir los datos ralativos a las bases de datos de origen y de destino y del FTP de destino:

    Datos generales de la clonación


  • Identificador de Entidad de Origen: Identificador de la entidad a clonar (obligatorio)
  • Identificador de Entidad de Destino: Identificador asignado a la nueva entidad (obligatorio)


    • Datos de la base de datos de origen


  • Dirección BBDD: Dirección IP o nombre de la máquina con la base de datos de origen (obligatorio)
  • Puerto BBDD: Puerto de trabajo de la base de datos (obligatorio)
  • Tipo de BBDD: Tipo de base de datos disponible (obligatorio)
  • Usuario: Nombre del usuario de base de datos (obligatorio)
  • Contraseña: Contraseña de la base de datos (obligatorio)
  • Repita contraseña: Repetición de la contraseña de la base de datos (obligatorio)


    • En caso de seleccionar como tipo de base de datos "Oracle" o "Db2", se deberá especificar la instancia con la que trabajar y los diferentes esquemas de conexión para cada uno de los módulos.

    Datos de la base de datos de destino


  • Dirección BBDD: Dirección IP o nombre de la máquina con la base de datos de destino (obligatorio)
  • Puerto BBDD: Puerto de trabajo de la base de datos (obligatorio)
  • Tipo de BBDD: Tipo de base de datos disponible (obligatorio)
  • Usuario: Nombre del usuario de base de datos (obligatorio)
  • Contraseña: Contraseña de la base de datos (obligatorio)
  • Repita contraseña: Repetición de la contraseña de la base de datos (obligatorio)


    • En caso de seleccionar como tipo de base de datos "Oracle" o "Db2", se deberá especificar la instancia con la que trabajar y los diferentes esquemas de conexión para cada uno de los módulos. El tipo de base de datos de origen deberá coincidir con el de destino.

    Datos del FTP de destino


  • Dirección FTP: Dirección IP o nombre de la máquina con FTP de destino (obligatorio)
  • Puerto FTP: Puerto de trabajo del FTP (obligatorio)
  • Usuario: Nombre del usuario del FTP (obligatorio)
  • Contraseña: Contraseña del FTP (obligatorio)
  • Repita contraseña: Repetición de la contraseña del FTP (obligatorio)


    • Una vez introducidor todos los datos, pulse sobre el botón Comprobar. Si todos los datos son correctos y se ha podido establecer la conexión correctamente con la base de datos de origen y de destino y el FTP de destino, se habilitará el botón Aceptar. Si el usuario está de acuerdo con los datos introducidos pulse el botón Aceptar para iniciar el proceso de clonación.

    En caso de haber validado los datos y estos ser correctos, si se modifica alguno de los valores introducidos, el usuario deberá repetir el proceso de validación pulsando de nuevo el botón Comprobar.

    Tanto si el proceso ha sido correctamente iniciado como si no, el usuario será informado del estado actual.

    4) Una vez iniciado correctamente la clonación, el usuario podrá seguir al detalle el proceso pulsando sobre el botón de Monitorización

    El proceso de clonación se compone de dos procesos, una exportación y una importación. Por lo tanto, seleccione del desplegable de exportaciones el proceso de exportación iniciado (identificación del proceso mediante la hora de inicio). Una vez finalizado el proceso de exportación, se podrá monitorizar el proceso de importación seleccionando el proceso del desplegable de importaciones (identificación del proceso mediante la hora de inicio).

    En la parte superior se mostrará un aviso indicando si el proceso se encuentra en estado de ejecución o si ya ha finalizado. En la parte central de la pantalla se visualizará el log asociado al proceso seleccionado


    En la parte inferior, el usuario tiene la posibilidad de actualizar automáticante el log del proceso cada x segundos (introducir cada cuantos segundo se desea actualizar, con un valor mínimo de 10 segundos, y pulsar el botón Actualizar). Este proceso de actualización sólo se realizará si el proceso seleccionado no ha finalizado. En caso de querer detener el proceso de actualización automática, deje vacío el campo de tiempo y pulse sobre el botón Actualizar

    5) Para que el proceso de clonación se considere completo, se deberá realizar la configuración de base de datos para la nueva entidad en el fichero de configuración del servidor de aplicaciones mediante el botón Configuración del Servidor de Aplicaciones

    Datos para añadir la nueva entidad en el fichero de configuración del Servidor de Aplicaciones

  • Fichero de configuración: Fichero de configuración del Servidor de Aplicaciones (obligatorio)
  • Tipo de Servidor: Tipo del servidor de aplicaciones: Tomcat o JBoss (obligatorio)
  • Identificador de Entidad: Identificador de la nueva entidad a añadir (obligatorio)
  • Dirección BBDD: Dirección IP o nombre de la máquina con la base de datos (obligatorio)
  • Puerto BBDD: Puerto de trabajo de la base de datos (obligatorio)
  • Tipo de BBDD: Tipo de base de datos disponible (obligatorio)
  • Usuario: Nombre del usuario de base de datos (obligatorio)
  • Contraseña: Contraseña de la base de datos (obligatorio)
  • Repita contraseña: Repetición de la contraseña de la base de datos (obligatorio)


    • En caso de seleccionar como tipo de base de datos "Oracle" o "Db2", se deberá especificar la instancia con la que trabajar y los diferentes esquemas de conexión para cada uno de los módulos.

    Una vez introducidor todos los datos, pulse sobre el botón Comprobar. Si todos los datos son correctos y se ha podido establecer la conexión correctamente con la base de datos, se habilitará el botón Aceptar. Si el usuario está de acuerdo con los datos introducidos pulse el botón Aceptar para iniciar el proceso de configuración de la nueva entidad.



    El contenido del fichero de configuración modificado para la nueva entidad se podrá visualizar por pantalla. El usuario tiene la posibilidad de descargar el contenido pulsando sobre el botón Descargar. Es recomendable hacer una copia de seguridad del fichero de configuración del Servidor de Aplicaciones antes de sustituirlo por el nuevo fichero.
