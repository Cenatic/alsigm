Para que el proceso de importación o de clonación se considere completo, se deberá realizar la configuración de base de datos para la nueva entidad en el fichero de configuración del servidor de aplicaciones mediante el botón Configuración del Servidor de Aplicaciones. Este proceso se deberá realizar siempre que se importe o se clone una entidad.

Descripción de elementos:

Datos para añadir la nueva entidad en el fichero de configuración del Servidor de Aplicaciones

  • Fichero de configuración: Fichero de configuración del Servidor de Aplicaciones (obligatorio)
  • Tipo de Servidor: Tipo del servidor de aplicaciones: Tomcat o JBoss (obligatorio)
  • Identificador de Entidad: Identificador de la nueva entidad a añadir (obligatorio)
  • Dirección BBDD: Dirección IP o nombre de la máquina con la base de datos (obligatorio)
  • Puerto BBDD: Puerto de trabajo de la base de datos (obligatorio)
  • Tipo de BBDD: Tipo de base de datos disponible (obligatorio)
  • Usuario: Nombre del usuario de base de datos (obligatorio)
  • Contraseña: Contraseña de la base de datos (obligatorio)
  • Repita contraseña: Repetición de la contraseña de la base de datos (obligatorio)


    • En caso de seleccionar como tipo de base de datos "Oracle" o "Db2", se deberá especificar la instancia con la que trabajar y los diferentes esquemas de conexión para cada uno de los módulos.

    Una vez introducidor todos los datos, pulse sobre el botón Comprobar. Si todos los datos son correctos y se ha podido establecer la conexión correctamente con la base de datos, se habilitará el botón Aceptar. Si el usuario está de acuerdo con los datos introducidos pulse el botón Aceptar para iniciar el proceso de configuración de la nueva entidad.



    El contenido del fichero de configuración modificado para la nueva entidad se podrá visualizar por pantalla. El usuario tiene la posibilidad de descargar el contenido pulsando sobre el botón Descargar. Es recomendable hacer una copia de seguridad del fichero de configuración del Servidor de Aplicaciones antes de sustituirlo por el nuevo fichero.
