Componentes >> Entidades Atrás

Para la parametrización de procedimientos administrativos se cuenta con Entidades, que permiten almacenar información estructurada de los distintos expedientes.

Existen una serie de entidades precargadas que son comunes a todos los procedimientos, aunque adicionalmente se pueden crear nuevas entidades para cubrir los datos específicos de de cada caso.

En el catálogo de Entidades se muestran todas las entidades dadas de alta en el sistema.

Al crear una entidad desde el catálogo de procedimientos se creará una tabla en base de datos, con los campos introducidos por el usuario, más una serie de campos de control requeridos por el sistema.

Pulsando sobre el nombre de la entidad, se accede a las propiedades de la misma:

  • Datos generales de la entidad:


    • Nombre: Nombre que se utilizará en invesFlow para hacer referencia a la entidad.
    • Tabla BD
      • : Indica el nombre de la tabla en la base de datos.
    • Descripción: Objetivo de la entidad.


  • Campos de la entidad: en esta ventana se visualizan los distintos campos de la entidad. Para crear uno nuevo pulsar la opción “Añadir”.

    • Etiqueta
      • : Nombre que tendrá el dato en los formularios
    • Columna BD
      • : Nombre que tendrá la columna en la base de datos.
    • Descripción
      • : Objetivo del campo.
    • Tipo
      • : Tipo de campo, seleccionable de una lista de valores disponibles.
    • Tamaño
      • : Tamaño del campo, en caso de seleccionar un campo de tipo texto o decimal
    • Precisión
      • : Precisión del campo, en caso de seleccionar un campo de tipo decimal


  • Índices de entidad: si se desea se pueden crear índices para algunos de los campos creados en el apartado anterior. Será interesante crear un índice sobre un campo cuando este se utilice mucho en las búsquedas y esté previsto que el volumen de datos de la entidad sea elevado.

    • Clave única
      • : indica que el valor de un campo no se pueda repetir.


  • Validaciones de la entidad: se utiliza esta opción cuando el valor de un campo está validado contra una serie de valores normalizados, en lugar de permitir que el usuario lo introduzca a mano. La validación se realiza contra tablas de validación, creadas y mantenidas en el propio sistema:

    • Tabla
      • : se indica la tabla sobre la que se realizará la validación

    Se ha de indicar además si es obligatorio que el campo tenga algún valor, o si por el contrario puede estar vacío.


  • Formularios de la entidad:Los formularios son empleados en el tramitador para que el usuario visualice e introduzca datos para una o varias entidades.

    Un formulario inicialmente tiene una entidad principal, y se crea asociado a la misma, aunque puede combinar datos de varias de ellas.

    Pulsando sobre el nombre de un formulario se pueden ver las propiedades de ésta:

    • Nombre
      • : nombre con que se identificará el formulario

    • Descripción
      • : nombre que verá el usuario en el tramitador de expedientes en la pestaña correspondiente al formulario.

    • Clase
      • : este campo viene cumplimentado por el sistema, aunque se permite cambiar. Indica la clase que va a implementar el formulario.

    • Parámetros
      • : Se pueden indicar, por ejemplo, entidades secundarias para el formulario o validaciones sobre los campos de la entidad, necesarias para la clase definida anteriormente.

      Por defecto vienen definidas las validaciones usadas en los campos validados de la entidad, para que en la visualización del formulario se muestren los controles de validación correspondientes.

    • Formateador
      • : este campo permite definir qué campos se van a visualizar cuando se muestren listas de entidades. Las listas de entidades se muestran cuando la relación de la entidad con el expediente es de n a 1.

    • Ubicación en servidor
      • : Indica la ruta donde se almacena el formulario en el servidor; se deberá modificar este valor si se desea almancenarlo en otra ubicación.

    Al crear una entidad, automáticamente se crea un formulario por defecto o formulario de entidad que muestra todos los campos de esta, y que se podrá utilizar desde los procedimientos.

    Todo formulario se puede modificar, y se pueden crear cuantos formularios se deseen por entidad; que se basarán en el 'formulario de entidad' creado por defecto.

    Si se hacen cambios en la estructura de la entidad (nuevos campos o validaciones), se deberá regenerar el formulario de la entidad, y posteriormente regenerar los formularios creados a partir del mismo; si se han hecho cambios de diseño o código se perderán, por lo que se recomienda guardar el diseño y/o código de los mismos para volver a introducir los cambios en los formularios regenerados.

    Para modificar un formulario, desde la lista de formularios de una entidad, se pulsa sobre la ruta del formulario. Se mostrará una ventana que permite descargar el formulario, tanto su diseño .html como su código .jsp, a un directorio local.

    Sobre el diseño del formulario en formato .html, se podrá editar y modificar con cualquier editor de formularios, el aspecto de los campos, incluso introducir nuevos elementos, pero no se pueden cambiar los nombres ni tipos de los campos existentes.
    Igualmente se podrá modificar el código .jsp, para por ejemplo, introducir nuevos controles en el formulario, o validaciones javascript.

    Una vez modificado el formulario, se puede actualizar en el catálogo desde la misma ventana que se descargó. Se pulsará “Examinar” para localizar el formulario en el disco duro local, y “Actualizar” para subirlo al servidor y sustituir el existente. El sistema notificará con una alarma si se han modificado datos de la estructura y no puede actualizar correctamente el formulario.

    Una vez hechos los cambios, el formulario estará disponible para ser usado en el tramitador.
